What?

CSS grid layout helps to divide page or elements in aligned areas.



This CSS style helps to align elements vertically and horizontally.
It looks like `<table>` but more architecture oriented.

When is it best to use?

TechniqueWhen to use it
<table>To present data
FloatTo move an element apart of the current flow (like a figure in an article)
position: absoluteTo display a unique element at a specific position (but this should be avoided when it is possible)
FlexTo display a 1D list of elements (and mainly when the number of elements can changed)
GridTo structure elements in 2D

Where?

Compatibility

CSS grid layout are part of CSS3 features and so are available on modern browser.

IE10 (until Edge 15) can support most of features with the -ms- prefix.



Some attributes or properties have been changed recently by the W3C and so are not yet available in browser under their correct form (but works with the grid- prefix).
All code shown here, follow the standard (as defined in 2018).

How?

CSS attributes can be set on the container element (this is mainly to define the grid) and on content elements (this is to place the element on the grid).

On container

display: grid

display: grid

This is the main attribute to set on a container element. This element becomes a grid-container and its child become grid-items.

Example

.grid {
    display: grid;
    grid-template: 40% 60% / 20% 60% 20%;
}

On container

grid-template

This is to define the grid. Its number of cells and their size.

grid-template-rows

This defines the size of each row.


Syntax (simplified)

grid-template-rows: <height-value> <height-value> ...

height-value can be any value with a unit or a percent

Example

                .grid {
                    display: grid;
                    grid-template-rows: 50px 2em 50%;
                }

grid-template-columns

This defines the size of each column.


Syntax (simplified)

grid-template-columns: <width-value> <width-value> ...

width-value can be any value with a unit or a percent

Example

.grid {
    display: grid;
    grid-template-columns: 150px 5em 50%;
}

grid-template

This is a shorthand for grid-template-rows and grid-template-columns.


Syntax

grid-template-columns: <grid-template-rows-value> / <grid-template-columns-value>

grid-template-rows-value is any value we can use with grid-template-rows
grid-template-columns-value is any value we can use with grid-template-columns

Example

.grid {
    display: grid;
    grid-template: 2em 130px / 150px 5em 50%;
}

Special functions, values and units

With grid layers there are some several functions, values and units which greatly help.

Somes are specific for grid layers. And somes are just good to know that they work great with grid layers (and they can also be used with Multi-column or Flexible Box).

Unit: fr

fr for "fraction" (or free space) is a unit about remaining spaces.

If several values use the fr unit, they share the remaining spaces.


Considering a grid container having a width of 100px and containg a column of 30px. The remaining space is worth 70px.

  • if one column has value 1fr, its width is 70px.
  • if two columns have value 1fr, their width are 35.5px for both.
  • if one columns has value 1fr and a second one 4fr, the first column's width is 14px and the second column's width is 56px.

Example

.grid {
    display: grid;
    grid-template: 40% 1fr / 2fr 4em 1fr;
}

Function: repeat

repeat is to repeat the same instructions several times.

Syntax

repeat(<nb occurrence>, <value>)

nb occurrence is the number of time to repeat the instruction.
value is any value to repeat.

Example

.grid {
    display: grid;
    grid-template: repeat(3, 1fr) 2fr / 1fr repeat(2, 40%);
}

Value: auto-fill

Using with repeat, it is worth the number of occurrence needed to fill the grid.


Example

                            repeat(auto-fill, 10%);

In this example, auto-fill is worth 10


Value: auto-fit

Behaves like auto-fill but it also removes empty tracks.

Example

.grid {
    display: grid;
    grid-template: repeat(auto-fill, 25%) / 3em repeat(auto-fill, 20%);
}

Function: minmax

minmax is to find a value which is in the given min-max range. The value cannot be lesser than minimul value and cannot be greater than maximum value.

This is a little bit similar to min-width and max-width.

Syntax

minmax(<min-value>, <max-value>)

min-value is the minimal value and can be of any value type.
max-value is the maximal value and can be of any value type.

Example

.grid {
    display: grid;
    grid-template: 1fr 1fr / 1fr minmax(200px, 40%);
}
width = 300px
width = 800px

width = 200px

width = 320px → 40%

Value: min-content and max-content

min-content is the value of the smallest size of all contents without breaking words (or elements).

In another words, this is the largest chunk of all contents.


max-content is the value of the largest size of all contents.

Example

.grid {
    display: grid;
    grid-template-rows: 1fr 1fr 1fr;
    grid-template-columns: min-content max-content min-content;
}
small text
small text
aTextWithoutWrappingContent
a more longer text
a more longer text
A text longer than previous ones but with smaller words.
small chunck
aLargerText
Some several small words.

In bold are the important elements which counts for the width of columns.

On container

Alignements

align-items

It sets the default alignements of direct children item within its containing block.
Basically it aligns items vertically.


Syntax

align-items: <position>
align-items: <overflow-position> <position>
align-items: <baseline-position> baseline
align-items: baseline

position can be any of normal, stretch, center, start, end, self-start, self-end, flex-start, flex-end

overflow-position can be safe or unsafe

baseline-position can be first or last

Position values

  • normal: similar to stretch, or start if item has an intrinsic sizes.
  • stretch: Items are stretched to fit the cell dimensions.
  • center: Items' margin boxes are centered within the line.
  • start: Items are aligned at the start edge of the alignment container.
  • end: Items are aligne at the end edge of the alignment container.
  • self-start: Items are aligned at the edge of the alignment container of the start side of the item.
  • self-end: Items are aligned at the edge of the alignment container of the end side of the item.
  • flex-start: The cross-start margin edges of the items are flushed with the cross-start edge of the line.
  • flex-end: The cross-end margin edges of the items are flushed with the cross-end edge of the line.

baseline

  • first baseline: Items are aligned on the first baseline of their container.
  • first baseline: Items are aligned on the last baseline of their container.

overflow position

  • safe: If the size of the items overflow the alignment container, the items are instead aligned as if the alignment mode were start.
  • unsafe: Regardless of the relative sizes of the items and alignment containers, the given alignment value is honored.

Example

.grid {
    display: grid;
    grid-template: 40% 60% / 20% 60% 20%;
    align-items: ;
}
A cell with a longer text

It sets the default alignements of direct children item within its containing block.
Basically it aligns items horizontally.


Syntax

justify-items: <position>
justify-items: <overflow-position> <position>
justify-items: legacy <left | center | right>
justify-items: <baseline-position> baseline
justify-items: baseline

position can be any of normal, stretch, center, start, end, self-start, self-end, flex-start, flex-end

overflow-position can be safe or unsafe

baseline-position can be first or last

legacy

It makes the value inherited by the box descendants.


If legacy is set any sub-grid will apply the following value by default.

Example

    display: grid;

    justify-items: ;
}

If only one value is set, it applies for both.

On container

grid-auto-flow

grid-auto-flow

Defines how the auto-placement algorithm works.


By default, elements are set inside the first rows by columns then it fills the next row.


Syntax

grid-auto-flow: <value>
grid-auto-flow: <value> dense
grid-auto-flow: dense

value is either "row" either "column".

Values

  • row: Fill row first.
  • column: Fill column first.

  • dense: If smaller item comes later, algorithm set it before to fill holes.
    when dense is not set all items are added in order (and so holes can appear).

Example

.grid {
    display: grid;
    grid-template: 40% 60% / 20% 60% 20%;
    grid-auto-flow: ;
}

On container

Auto generate rows and columns

grid-auto-rows

Defines the size of added rows (the ones which are not defined in templates).


Syntax

grid-auto-rows: <row-size> ...

row-size is any value to define the size of rows.

The value is a pattern.
So if there are several row-size. The first added row, will use the first value. Then it will use the next one. If there are no more value the first one is used again. etc...

Example

.grid {
    display: grid;
    grid-template: 2em 4em / repeat(3, 1fr);
    grid-auto-rows: 1em;
}
                            
.grid {
    display: grid;
    grid-template: 10fr / 1fr;
    grid-auto-rows: 20px 40px 60px;
}

grid-auto-columns

Defines the size of added columns (the ones which are not defined in templates).


Syntax

grid-auto-columns: <col-size> ...

col-size is any value to define the size of columns.

The value is a pattern.
So if there are several col-size. The first added column, will use the first value. Then it will use the next one. If there are no more value the first one is used again. etc...

Example

.grid {
    display: grid;
    grid-template: repeat(2, 1fr) / 30%;
    grid-auto-flow: column;
    grid-auto-columns: 4em;
}
                            
.grid {
    display: grid;
    grid-template: 2em 1fr / 3em;
    grid-auto-flow: column;
    grid-auto-columns: 30px 60px 90px;
}

On content

order

order

Defines the order to fill the grid. Elements with a higher order are added later to the grid.
If 2 elements have the same order, the oldest is added first.


By default, all elements have the order set to value 0.


Syntax

order: <value>

value is any integer number (which can be negative).

Example

.item1 { order: 5; }
.item2 { order: -1; }
.item3 { order: 4; }
.item4 { order: 5; }

On content

grid-row

grid-row-start

Defines at which rows the element starts.

It counts grid lines.
Rows start at 1.
Last grid lines (the end of the grid) can be identified by -1, (-2 is to put the item in the last row).


Syntax

grid-row-start: <value>

value is any integer number (which can be negative).

Example

.item1 { grid-row-start: 2; }
.item2 { grid-row-start: 1; }
.item3 { grid-row-start: -2; }

grid-row-end

Defines at which rows the element ends.

It counts grid lines.

To end at the first row, the correct index is 2.
To end on last row the index can be -1.


Syntax

grid-row-end: <value>

value is any integer number (which can be negative).

Example

.item1 { grid-row-end: 3; }
.item2 { grid-row-end: 2; }
.item3 { grid-row-end: -1; }

It is possible to combine grid-row-start and grid-row-end to have an element over several rows.

Example

.item1 { grid-row-start: 1; grid-row-end: 3; }
.item2 { grid-row-start: -2; grid-row-end: -1; }
.item3 { grid-row-start: 1; grid-row-end: -1; }

grid-row

grid-row is a shorthand for grid-row-start and grid-row-end.


Syntax

grid-row: <grid-row-start-value> / <grid-row-end-value>
grid-row: <grid-row-start-value>

grid-row-start-value is any accepted value by grid-row-start.
grid-row-end-value is any accepted value by grid-row-end.

Example

.item1 { grid-row: 1; }
.item2 { grid-row: 1 / 2; }
.item3 { grid-row: 1 / -1; }

grid-area

Defines area where the elements shoud be displayed.

grid-area: <grid-row-start-value> / <grid-column-start-value> / <grid-row-end-value> / <grid-column-end-value>

Special functions, values and units

Value: span

span helps to define the length of the elements (which can be used either for rows or for columns).


It can be used with grid-row-start, grid-row-end, grid-column-start, grid-column-end and all properties which use the same syntax (grid-row, grid-column, grid-area).


span is not working very well when the other bound use negative value.


Syntax

span <value>
<value> span

value is an integer which represents length in number of cell.

Example

.grid { grid-template: repeat(5, 1fr) / repeat(5, 1fr); grid-auto-flow: dense; }
.item1 { grid-column: 2 / span 2; }
.item2 { grid-row-end: 3 span; }
.item3 { grid-area: 3 / span 3 / span 2 / 6; }
.item4 { grid-column-start: 2 span; grid-column-end: 5; }

Custom identifier

(or named area)

grid-template-areas

on container

Defines names to areas.


Those areas can then be referenced by grid placement propertie.


Syntax

grid-template-areas: "<row-area-definition>" ...

row-area-definition are any alpha-numeric string to define the area name.

A row area definition is enclosed by double quotes, names are separated by space.


Example

grid-template-areas: "row1column1 row2column2" "row2column1 row2column2";

grid placement properties

grid-row-start, grid-row-end, grid-column-start, grid-column-end

(on content)

The grid placement properties can use the area name to reference a specific area


Syntax

grid-row-start: <name>
grid-row-end: <name>
grid-column-start: <name>
grid-column-end: <name>

name are the ones given in grid-template-area.

Example

.grid { grid-template-areas: "cell1 cell2 cell3" "lowerCell1 lowerCell2 lowerCell3"; }
.item1 { grid-row-start: cell2; }
.item2 { grid-row-end: lowerCell3; }
.item3 { grid-column-start: lowerCell3; }
.item4 { grid-column-end: cell2; }

Note: Those names are only reference to grid lines.
This is why even by referencing lowerCell3, item3 is on 1st row.

<name>-start and <name>-end

When setting area names, it creates several named-lines.


For a name foo, it also creates foo-start (which refers to the top line or the left line of the area depending of the property direction) and foo-end (which refers to the bottom line or the right line of the area depeding of the property direction).



By default, properties grid-row-start and grid-column-start read the *-start line name.

By default, properties grid-row-end and grid-column-end read the *-end line name.

Example

.grid { grid-template-areas: "cell1 cell2 cell3" "lowerCell1 lowerCell2 lowerCell3"; }
.item1 { grid-row-start: cell2-end; }
.item2 { grid-row-end: lowerCell3-start; }
.item3 { grid-column-start: lowerCell1-end; }
.item4 { grid-column-end: cell3-start; }

Shorthand

grid-row, grid-column, grid-area

(on content)

It is possible to use the shorthand properties.
When using a name area it will refers to both the *-start and *-end grid line.


Syntax

grid-row: <name> / <name>;
grid-row: <name>;
grid-column: <name> / <name>;
grid-column: <name>;
grid-area: <name> / <name> / <name> / <name>;
grid-area: <name> / <name>;
grid-area: <name>;

name are the ones given in grid-template-area.

Example

.grid { grid-template-areas: "cell1 cell2 cell3" "lowerCell1 lowerCell2 lowerCell3"; }
.item1 { grid-area: lowerCell2; }
.item2 { grid-column: cell2 / lowerCell3; }
.item3 { grid-row: cell2; }

Using the same name

If several cells contain the same name, the *-start grid line refers to the first occurrence and the *-end to the last.


It is possible to define large area like this.

Example

.grid { grid-template-areas: "header header menu" "footer footer menu"; }
.item1 { grid-area: menu; }
.item2 { grid-area: footer; }
.item3 { grid-area: header; }

unnamed area: .

If a cell should not have any specific name, it is possible to use the character .

Example

.grid { grid-template-areas: "header content content" ". content content"; }
.item1 { grid-area: content; }
.item2 { grid-area: header; }

grid-template

On container

It is also possible to use the shorthand grid-template to define areas while defining the grid.


Syntax (simplified)

grid-template: "<row-area-definition>" <row-size> ... / <column-size>...

row-area-definition are any alpha-numeric string to define the area name.

row-size and column-size are accepted values by grid-template-rows and grid-template-columns.

Example

.grid {
    grid-template: "header header header" 2em
                   "menu content content" 1fr
                   ". footer ." 2em
                   / 20% 1fr 20%;
}
.item1 { grid-area: content; }
.item2 { grid-area: header; }
.item3 { grid-area: menu; }
.item4 { grid-area: footer; }

Custom line grid

To create custom line grid, line names should be put inside [].

It is possible to define several line at the same place

All template properties accepts them (grid-template-rows, grid-template-columns, grid-template).


Syntax (simplified)

grid-template-rows: [<grid-line-name>] <height-value> [<grid-line-name>] ...
grid-template-columns: [<grid-line-name>] <width-value> [<grid-line-name>] ...
grid-template: [<grid-line-name>] <height-value> [<grid-line-name>] ... / [<grid-line-name>] <width-value> [<grid-line-name>] ...

grid-line-name can be any alpha-numeric string to define the grid line name.

height-value can be any value to define the height of rows.

width-value can be any value to define the width of columns.

Example

.grid {
    grid-template: [first-row-line] 2em
                   [middle-row-line1] 1fr
                   [middle-row-line2] 2em [last-row-line]
                   / [first-col-line] 20%
                     [middle-col-line1] 1fr
                     [middle-col-line2 alternate-name] 20% [last-col-line];
}
.item1 {
    grid-row-start: middle-row-line1;
    grid-row-end: last-row-line;
}
.item2 { grid-column: alternate-name / last-col-line; }
.item3 { grid-area: first-row-line / middle-col-line1 / middle-row-line2 / middle-col-line2; }

Example

Using both custom grid lines and custom area names.

.grid {
    grid-template: [first-row-line] ". content logo" 2em
                   [middle-row-line1] "menu content ." 1fr
                   [middle-row-line2] "menu . ." 2em [last-row-line]
                   / [first-col-line] 20%
                     [middle-col-line1] 1fr
                     [middle-col-line2] 20% [last-col-line];
}
.item1 {
    grid-row-start: middle-row-line1;
    grid-row-end: last-row-line;
}
.item2 { grid-column: logo; }
.item3 { grid-area: content-start / content-start / middle-row-line2 / middle-col-line2; }

Same grid line names

If several grid line have the same name and are in the same direction (row vs column), it is possible to refer to one of them by using an index number near it.

All placements properties accepts them (grid-row-start, grid-row-end, grid-row, grid-column-start, grid-column-end, grid-column, grid-area).


Syntax (simplified)

grid-row-start: <grid-line-name> <index>
grid-row-start: <index> <grid-line-name>
...

grid-line-name a reference to a grid line name.

index an integer to identify the correct grid line.
By default, it is worth 1.
It can be negative (-1 refers to the last grid line name).

index cannot be 0

Example

.grid {
    grid-template: [border-line] 2em
                   [row-line] 1fr
                   [row-line] 2em
                   [border-line]
                   / [border-line] 20%
                     [col-line] 1fr
                     [col-line] 20%
                     [border-line];
}
.item1 {
    grid-row-start: 1 row-line;
    grid-row-end: -1 border-line;
}
.item2 { grid-column: col-line 2 / border-line 2; }
.item3 { grid-area: border-line / col-line / row-line -1 / col-line -1; }

repeat

It is possible to create custom grid line with repeat.


Syntax

repeat(<nb occurrence>, [<grid-line-name>] <value>)
repeat(<nb occurrence>, <value> [<grid-line-name>])
repeat(<nb occurrence>, [<grid-line-name>] <value> [<grid-line-name>])

nb occurrence is the number of time to repeat the instruction.

value is any value to repeat.

grid-line-name is any grid line to repeat.

Example

.grid {
    grid-template: repeat(auto-fill, [row-line] 2em) / repeat(auto-fill, 20% [col-line]);
}
.item1 {
    grid-row-start: 2 row-line;
    grid-row-end: -2 row-line;
}
.item2 { grid-column: before-col -2 / col-line -1; }
.item3 { grid-area: row-line / col-line / row-line 5 / col-line 3; }

Grid

The grid property is a shorthand for grid-template and grid-auto-rows/grid-auto-columns.


Syntax

grid: <grid-template-rows-value> / <grid-auto-columns-value>;
grid: <grid-auto-rows-value> / <grid-template-columns-value>
grid: <grid-template-value>

grid-template-rows-value can be any value of grid-template-rows

grid-template-columns-value can be any value of grid-template-columns

grid-auto-rows-value can be any value of grid-auto-rows

grid-auto-colums-value can be any value of grid-auto-columns

grid-template-value can be any value of grid-template

Example

.grid {
    grid: repeat(2, 50%) / auto-flow 25%;
}
                            
.grid {
    grid: auto-flow 1fr / repeat(3, 1fr);
}
