Qu'est-ce que le JavaScript?

(ou plus souvent appelé JS)

Principes

Le JavaScript est un langage de programmation. Il permet d'exécuter des séquences de codes.


Il s'agit de faire marcher un script qui est interprété au moment de l'exécution. Il n'est donc pas compilé en source binaire contrairement à d'autres langages.


Il n'a rien à voir avec le Java qui est un autre langage de programmation.

Exécution

Il faut un interpréteur pour traduire le script en code compréhensible par l'ordinateur.


Les interpréteurs les plus utilisés sont ceux intégrés dans les navigateurs (Chrome, Firefox, Safari, Opera, Edge, ...). Mais il existe aussi d'autres interpréteurs (comme NodeJs) qui permettent d'exécuter du JS directement sur l'ordinateur.


L'avantage de l'interpréteur intégré aux navigateurs, c'est qu'il permet de manipuler simplement un document HTML et donc d'obtenir facilement une interface pour l'utilisateur.
Les pages sur internet sont des fichiers HTML qui sont exposées publiquement.

Premiers pas en JavaScript

Première exécution

Les explications arriveront un peu plus tard ☺.

  • Ouvrez un éditeur de texte et entrez le code suivant :
    
<html>
  <body>
    <script>
alert('Bonjour');
    </script>
  </body>
</html>
  • Enregistrez ce code dans un fichier nommé bonjour.html.
  • Ouvrez ce fichier dans votre navigateur préféré.
  • Une fenêtre avec "Bonjour" est affichée.

Le code JavaScript correspond aux lignes entre <script> et </script>

La fonction alert est une fonction de debug, et ne doit normalement pas être utilisée.

Vous pourrez utiliser ce fichier, ou une copie de celui-ci, pour tester les instrutions présentées ensuite.

Principes d'exécution

Les instructions sont exécutées les unes après les autres de haut en bas.


Généralement, une instruction tient sur une ligne et se termine par un ; (en réalité il est possible de couper une instruction sur plusieurs lignes, mais nous n'en parlerons pas dans cette présentation).

Variables

Une variable permet d'enregistrer une valeur. Il sera possible de la réutiliser plus tard.


Il existe différent type de variables mais en JavaScript on les déclare toutes de la même manière avec un des mots clefs suivant var, let ou const.


Nous expliquerons les différences entre ces mots clefs plus tard.

Nommer une variable

Un nom de variable est composé de n'importe quels caractères alphabétique, numérique et les caractères _ et $.
Mais une variable ne peut pas débuter par un caractère numérique.

Il est fortement conseillé de donner un nom de variable explicite pour comprendre à quoi sert cette variable (et il est assez courant d'utiliser des noms en anglais).

Type: Booléen (Boolean)

Une variable de type booléenne possède 2 valeurs possible: true (vraie) ou false (fausse).


var variable1 = true;
var variable2 = false;

Type: Chaîne (String)

Une variable de type string représente une chaîne de caractères. Une chaîne de caractères est délimitée par les caractères ', " ou `; et doit se terminer avec le même caractère.

Il n'y a pas de différence entre les chaînes construite avec '' et "".
Nous n'aborderons pas les différences des chaînes délimitées par ``.

Pour échapper un caractère (par exemple le caractère de délimitation, il faut le précéder par le caractère \ (et lui même doit-être échappé si l'on souhaite l'utiliser)).


var variable1 = 'chaine 1';
var variable2 = "chaine 2";
var variable3 = `chaine 3`;

var exemple1 = 'c\'est une chaîne qu\'il faut "échapper"';
var exemple2 = "c'est une chaîne qu'il faut \"échapper\"";

Type: Nombre (Number)

Une variable de type number représente un nombre. Celui-ci peut être positif ou négatif, entier ou décimal.


var variable1 = 42;
var variable2 = -234;
var variable3 = 1234.56789;

Type: Indéfini (undefined)

Une variable peut être de type undefined si elle n'a jamais été défini (aucune valeur ne lui a été assignée).


var variable1;
var variable2 = undefined;

Il est possible d'utiliser le mot clef undefined pour mettre une variable dans cet état. Toutefois il faut faire attention à ne jamais écraser cette valeur (en créant une variable nommée undefined, ce qui est une très mauvaise pratique).

Débugger

Avant de se lancer dans le code, nous allons voir comment débugger le code.


Cela permettra de tester des fonctions simples et vous donnera la possibilité plus tard de comprendre ce qui ne marche pas

Outils de debugage

Tous les navigateurs récents possèdent des outils d'aide au débugage.

Vous pouvez ouvrir ces outils en utilisant la touche F12 ou ctrl + I


Plusieurs onglets sont disponibles. Ceux qui vont nous intéresser s'appellent (sous Chrome, pour les autres navigateurs les noms peuvent changer légèrement) Elements, Console, Sources et Network. Les autres ongletssont un peu plus complexe à utiliser et sortent du cadre de cette présentation.

Présentations des onglets

Elements

Vous pouvez observer tous les éléments du DOM (le résultat du HTML) dans cet outil. Il est également possible de modifier le style de chaque élément à la volée pour observer le résultat souhaité.

Network

Vous pouvez observer toutes les requêtes effectuées par le navigateur depuis cette page. Cela sert à vérifier les demandes faites au serveur leurs résultats.

Sources

Cet outil permet de voir le code source (le JS) tel qu'il sera exécuté. Il est possible de mettre des points d'arrêts (breakpoint), en cliquant sur le numéro de ligne. L'interpréteur s'arrêtera lorsqu'il rencontrera cette ligne, et vous pourrez observer la valeur de toutes les variables à cet instant.

Console

Cet outil permet d'exécuter du code JavaScript directement dans le navigateur. À noter que tous les résultats de l'API console se trouveront dans cet espace.

Api: console

console.log

Il est possible d'envoyer un message à la console avec la fonction console.log().

À noter qu'il est possible d'envoyer plusieurs arguments en même temps. console.log('message 1', 'message 2')


var test = 42;
console.log('résultat de la variable:', test);

Si vous testez ce code vous devriez voir s'afficher résultat de la variable: 42


Il existe aussi les méthodes info, warn et error qui permettent de marquer les messages différement et la console pourra les trier plus facilement.


var test = false;
console.info('résultat de la variable:', test);

Commentaires

Les commentaires permettent de mieux comprendre le but des lignes de code. Elles ne servent à rien pour l'ordinateur.

Il est très important d'en mettre à fin de pouvoir relire son code plus tard surtout quand celui-ci est devenu très gros.

Commentaire de ligne

Il s'écrit //. Tout ce qui suit sur la même ligne sera ignoré.


// texte ignoré
console.log('affiché'); // texte ignoré aussi

Commentaire de bloc

Il débute par /* et se finit par */. Tout ce qui est entre sera ignoré.


/* texte ignoré */
console.log('affiché'); /* texte ignoré
texte ignoré aussi
*/

Opérateurs de base

Un opérateur est une opération sur une ou plusieurs opérandes.

Cela permet de combiner des variables et réaliser des opérations entre elles.

Opérateur d'assignation

Opérateur: =

Opération: La valeur de droite est assignée à la variable de gauche.


var a = 34;
var b;
b = 42; // b vaut maintenant 42
b = a; // b vaut maintenant 34

Opérateur de type

Opérateur: typeof

Opération: Retourne le type de l'opérande. Le résultat est une string.

Il n'y a qu'une opérande à droite


var a = 12;
var b = '12';
var c;
console.log(typeof true); // 'boolean'
console.log(typeof a); // 'number'
console.log(typeof b); // 'string'
console.log(typeof c); // 'undefined'

Opérateur de négation

Opérateur: !

Opération: Converti l'opérande en boolean et retourne son opposé.

Il n'y a qu'une opérande à droite


var a = true;
var b = 1;
var c = 'text';
console.log(!false); // true
console.log(!0); // true
console.log(!''); // true
console.log(!a); // false
console.log(!b); // false
console.log(!c); // false

Opérateur de comparaison

Opérateur: == et ===

Opération: Retourne true si les deux opérandes sont identiques et false sinon.

L'opérateur === vérifie en plus que les types des opérandes sont identiques. Alors que == convertie les 2 opérandes dans le même type avant de comparer.


var a = 34;
var b = '34';
console.log(true === false ); // false
console.log(12 === 12); // true
console.log('texte' === 'texte'); // true
console.log(a === 34); // true
console.log(a === b); // false
console.log(a == b); // true

Il est conseillé de préférer l'usage de === à == car la comparaison est plus stricte.

Attention à ne pas les confondre avec =.

Opérateur de différence

Opérateur: != et !==

Opération: Retourne true si les deux opérandes sont différents et false sinon.

L'opérateur !== vérifie en plus les types des opérandes. Alors que == convertie les 2 opérandes dans le même type avant de comparer.


var a = 34;
var b = '34';
console.log(true !== false ); // true
console.log(12 !== 12); // false
console.log('texte' !== 'texte'); // false
console.log('texte' !== 'toto'); // true
console.log(a !== 34); // false
console.log(a !== b); // true
console.log(a != b); // false

Il est conseillé de préférer l'usage de !== à != car la comparaison est plus stricte.

Opérateur de comparaison plus petit que

Opérateur: <

Opération: retourne true si le premier nombre est plus petit que le deuxième. Retourne false sinon


console.log(10 < 20); // true
console.log(10 < -20); // false
console.log(10 < 10); // false

Opérateur de comparaison plus grand que

Opérateur: >

Opération: retourne true si le premier nombre est plus grand que le deuxième. Retourne false sinon


console.log(10 > 20); // false
console.log(10 > -20); // true
console.log(10 > 10); // false

Opérateur de comparaison plus petit que ou égal

Opérateur: <=

Opération: retourne true si le premier nombre est plus petit que le deuxième ou égal à celui-ci. Retourne false sinon


console.log(10 <= 20); // true
console.log(10 <= -20); // false
console.log(10 <= 10); // true

Opérateur de comparaison plus grand que ou égal

Opérateur: >=

Opération: retourne true si le premier nombre est plus grand que le deuxième ou égal à celui-ci. Retourne false sinon


console.log(10 >= 20); // false
console.log(10 >= -20); // true
console.log(10 >= 10); // true

Opérateur d'addition

Opérateur: +

Opération: Réalise la somme des 2 valeurs.


var a = 28;
var b = 14;
console.log(1 + 2); // 3
console.log(a + 5.6); // 33.6 et la variable a est inchangée
console.log(a + b); // 42

Attention si une des opérandes est une string alors l'opérateur devient celui de concaténation

Opérateur de soustraction

Opérateur: -

Opération: La valeur de droite est retirée à celle de gauche.


var a = 34;
var b = 12;
console.log(5 - 2); // 3
console.log(5 - 15); // -10
console.log(a - 5.6); // 28.4 et la variable a est inchangée
console.log(a - b); // 22

Opérateur de multiplication

Opérateur: *

Opération: Réalise la multiplication des 2 valeurs.


var a = 5;
var b = 12;
console.log(3 * 2); // 6
console.log(5 * -2); // -10
console.log(a * 2.5); // 12.5 et la variable a est inchangée
console.log(a * b); // 60

Opérateur de division

Opérateur: /

Opération: Réalise la division de la première opérande par la deuxième.


var a = 10;
var b = 4;
console.log(55 / 5); // 11
console.log(21 * -7); // -3
console.log(a / 5); // 2 et la variable a est inchangée
console.log(a / b); // 2.5

Opérateur modulo

Opérateur: %

Opération: retourne le reste de la division euclidienne de la première valeur par la deuxième.


console.log(4 % 10); // 4
console.log(18 % 10); // 8
console.log(10 % 10); // 0
console.log(8 % 5); // 3
console.log(12345 % 2); // 1

Opérateur de concaténation

Opérateur: +

Opération: Concaténe (regroupe) deux chaînes de caractères.


var a = 'alpha';
var b = 'beta';
console.log('a' + 'b'); // 'ab'
console.log(a + ' test'); // 'alpha test' et la variable a est inchangée
console.log(a + b); // 'alphabeta'

Attention il s'agit du même caractère + qui sert pour l'addition et pour la concaténation.
L'opération est différente en fonction des types des opérandes. 


console.log('1' + '2'); // '12'
console.log(1 + 2); // 3

Opérateur logique: ET

Opérateur: &&

Opération: retourne la première opérande si elle est équivalente à false sinon retourne la 2e opérande.

Cela correspond à un opérateur ET logique (retourne vraie si les 2 valeurs sont vraies).

Cet opérateur ne retourne pas forcément un booléen.


var a = 23;
var b = 42;
console.log(true && false); // false
console.log(true && true); // true
console.log(false && true); // false
console.log(0 && 12); // 0
console.log(a && 0); // 0
console.log(a && b); // 42

Opérateur logique: OU

Opérateur: ||

Opération: retourne la première opérande si elle est équivalente à true sinon retourne la 2e opérande.

Cela correspond à un opérateur OU logique (retourne vraie si l'une des 2 valeurs est vraie).

Cet opérateur ne retourne pas forcément un booléen.


var a = 23;
var b = 42;
console.log(true || false); // true
console.log(false || true); // true
console.log(false || false); // false
console.log(0 || 12); // 12
console.log(a || 0); // 23
console.log(a || b); // 23

Cet opérateur est souvent utilisé pour initialiser des variables avec une valeur par défaut.


var toto = a || b;

Priorités

Si on enchaîne plusieurs opérateurs à la suite, il s'exécuteront selon leur ordre de priorité


5 + 5 * 5 // vaut 30 (et non pas 50). La multiplication est plus prioritaire que l'addition

Pour changer l'ordre de priorité, il est possible d'utiliser des parenthèses ()


(5 + 5) * 5 // vaut 50 (l'opération dans les parenthèses est réalisée avant la multiplication)

Conversion de types

JavaScript est un langage typé faiblement.


C'est à dire qu'il est possible de changer les types des variables et de réaliser des opérations avec des variables de types différents.


Nous avons vu lors de quelques opérateurs que des conversions entre les types peuvent se produire implicitement. Nous allons voir ici le principe de ces conversions.

Conversion vers Number

Type d'origineRésultat
UndefinedNaN
Boolean
  • false → 0
  • true → 1
String
  • '' → 0
  • chaîne représentant un nombre (par exemple '-12.3') → le nombre représenté
  • chaîne possédant des caractères (par exemple '12px') → NaN

NaN (Not A Number) est un nombre qui représente une valeur qui n'est pas un nombre.

  • tous les autres → true

  • '' → false
  • toutes les autres (y compris '0') → true

Fonctions

Les fonctions sont des morceaux de code qu'on peut réutiliser.


Il est possible d'utiliser des arguments afin de modifier le résultat des fonctions.


Une fonction doit d'abord être déclarée, puis ensuite être appelée. Elle peut être appelée plusieurs fois.

Déclaration

Une fonction est déclarée en utilisant le mot clef function


// création d'une fonction appelée test1
function test1() {
    console.log('la fonction test1 a été appelée');
}

// assignation d'une fonction à une variable nommée test2
var test2 = function() {
    console.log('la fonction test2 a été appelée');
};

Le code situé entre les caractères { et } ne sera appelé que lorsque la fonction est appelée.

Appel

Une fonction est une variable de type "function".

On peut appeler une fonction en ajoutant () derrière le nom de la fonction qu'on souhaite appelé.


test1();
test2();

Retour d'une fonction

Une fonction peut retourner une valeur avec le mot clef return.

Lorsque l'interpréteur rencontre ce mot clef, il met fin à la fonction.


function test1() {
    return 'valeur retournée';
}

console.log(test1()); // affiche "valeur retournée"

Arguments

Les arguments sont des variables interne à la fonction, leurs valeurs initiales sont définies à l'extérieur de celle-ci.


function somme(a, b) {
    return a + b;
}

console.log(somme(1, 4)); // 5
console.log(somme(30, 8)); // 38

Portée des variables

Une variable déclarée à l'intérieur d'une fonction n'est utilisable qu'à l'intérieur de celle-ci.


var a = 1;
function test() {
    var a = 2;
    console.log(a); // affiche 2
}
test();
console.log(a); // affiche 1

Portée des variables (suite)

Si une variable n'est pas déclarée dans la fonction, elle est recherchée dans la fonction parente (et ainsi de suite).


var a = 1;
function test() {
    a = 2;
    console.log(a); // affiche 2
}
test();
console.log(a); // affiche 2

mini-exercice

Fil rouge: gestion de comptes

Simuler la gestion d'un compte

Le but sera de simuler la gestion d'un compte en effectuant des opérations d'ajout et de retrait d'argent du compte.

On souhaite:

  • Savoir le montant restant sur le compte (à l'aide d'une variable)
  • Savoir le nombre d'opérations réalisées (à l'aide d'une autre variable)
  • Effectuer une opération d'ajout (à l'aide d'une fonction)
  • Effectuer une opération de retrait (à l'aide d'une autre fonction)

Il n'est pas demandé au programme d'appeler ces fonctions tout de suite (elles pourront être appelées via la console)

Les montants d'ajout ou de retrait seront défini lors de l'appel des fonctions

En informatique, il n'existe jamais une seule solution. Il existe plein de manières différentes de réaliser la même tâche.
D'où l'intérêt de tester son programme.

Les solutions présentées sont une possibilité. Le fait que votre code soit différent ne veut pas dire qu'il n'est pas bon.


En supposant, que votre variable de compte s'appelle compte et vaut 1000 au départ.
Que votre variable de comptage d'opération s'appelle nbOperation.
Que vos fonctions d'opérations s'appellent ajout et retrait;

En exécutant le code ci-dessous après le votre, vous devriez obtenir les même valeurs qu'annoncées (sinon il y a un problème dans votre code).


ajout(500);
console.log(compte); // affiche 1500
retrait(200);
console.log(compte); // affiche 1300
console.log(nbOperation); // affiche 2

Exemple de solution


var compte = 1000; // création du compte
var nbOperation = 0; // nombre d'opérations réalisées

function affichageCompte() {
    console.log('Sur le compte il reste: ', compte);
    console.log('Il y a eut ' + nbOperation + ' opérations.');
}

function ajout(montant) {
    compte = compte + montant;
    nbOperation = nbOperation + 1;
    affichageCompte();
}

function retrait(montant) {
    compte = compte - montant;
    nbOperation = nbOperation + 1;
    affichageCompte();
}

Conditions

if

On peut choisir d'exécuter seulement certaines lignes de code en fonction de certaines condition.

Pour cela on utilise le mot clef if suivit de la condition (entourée de parenthèses) et suivit d'accolade {} qui contiendront le code a exécuter si la condition est vraie.


if (true) {
    console.log("le code est exécuté");
}
if (false) {
    console.log("le code est n'est pas exécuté");
}

else

On peut exécuter un autre bloc de code dans le cas ou la condition est fausse.

Pour cela on utilise le mot clef else après le bloc du if.


var a = 5;
if (a < 0) {
    console.log('a est négatif');
} else {
    console.log('a est positif (ou vaut 0)');
}

mini-exercice

Fil rouge: gestion de comptes

En reprenant l'exercice précédent.

  • Nous voudrions vérifier que les arguments passées en fonction sont bien de type number (sinon on ne fait rien).
  • Nous voudrions faire une seule fonction pour les 2 opérations d'ajout et de retrait avec le premier argument qui est une chaine valant soit 'ajout' soit 'retrait'.

Pour l'instant, nous considérerons que le premier argument ne peux avoir que ces 2 valeurs possible ('ajout' ou 'retrait')

Vérification

En supposant, que votre nouvelle fonction s'appelle operation.

En exécutant le code ci-dessous après le votre, vous devriez obtenir les même valeurs qu'annoncées (sinon il y a un problème dans votre code).


operation('ajout', 500);
console.log(compte); // affiche 1500
operation('retrait', 200);
console.log(compte); // affiche 1300
console.log(nbOperation); // affiche 2
operation('ajout', 'hahaha');
console.log(compte); // affiche 1300
console.log(nbOperation); // affiche 2

Exemple de solution


var compte = 1000; // création du compte
var nbOperation = 0; // nombre d'opérations réalisées

function affichageCompte() {
    console.log('Sur le compte il reste: ', compte);
    console.log('Il y a eut ' + nbOperation + ' opérations.');
}

function operation(action, montant) {
    if (typeof montant !== 'number') {
        return; // si le montant n'est pas un nombre on rejette l'opération
    }
    if (action === 'ajout') {
        compte = compte + montant;
    } else {
        compte = compte - montant;
    }
    nbOperation = nbOperation + 1;
    affichageCompte();
}

Boucles

Les boucles permettent de répéter plusieurs fois les mêmes lignes de code.

while

Les boucles while permettent de répéter un code tant que la condition donnée est vraie.

On commence par écrire le mot clef while suivit de la condition entre parenthèses, puis du bloc d'instructions à répéter.

Attention à bien écrire une condition qui deviendra fausse au bout d'un certain temps.
Sinon la boucle ne s'arrêtera jamais.


var compteur = 0;
while (compteur < 10) {
    compteur = compteur + 1;
    console.log(compteur);
}
/* Il va s'afficher les nombres de 1 à 10 */

for

Les boucles for permettent de créer des boucles incrémentales.

On commence par écrire le mot clef for suivit (entre parenthèses) de 3 instructions séparées par un ;: l'initialisation, la condition et l'incrémentation.

  • initialisation: cette instruction est appelée 1 seule fois au début de la boucle
  • condition: cette instruction est vérifiée avant chaque itération. Si elle est fausse, la boucle s'arrête
  • incrémentation: cette instruction est appelée à chaque fin d'itération

var compteur;
for (compteur = 1; compteur <= 10; compteur = compteur + 1) {
    console.log(compteur);
}
/* Il va s'afficher les nombres de 1 à 10 */

Portée des variables dans les blocs

Les variables déclarées avec var sont visible dans toute la fonction et sont donc visible à l'extérieur du bloc.

Les variables déclarées avec let ou const ne sont visible que dans le bloc.

La seule différence entre let et const, c'est qu'une variable déclarée avec const ne peut pas être modifiée.


var a = 1;
let b = 2;
if (true) {
    a = 10;
    b = 20;
    var c = 100;
    let d = 200;
}
console.log(a); // 10
console.log(b); // 20
console.log(c); // 100
console.log(d); // Erreur
                            

var a = 1;

  • Si la valeur de répétition n'est pas donnée, elle vaut 1

Objets

Les objets sont des variables de type "object".

Ils permettent de regrouper plusieurs variables en une seule entitée.

Création simple

Il est possible de créer un objet en utilisant les caractères {}.

Les propriétés de l'objet sont crées en mettant le nom de la propriété suivit de : puis de sa valeur.

Les propriétés sont séparées par le caractère ,


var objet1 = {};
var objet2 = {
    attribut1: true,
    attribut2: 'chaine',
    attribut3: 100,
    attribut4: function() {return 42;},
};

Utiliser les propriétés

Nous pouvons accéder aux propriétés d'un objet en mettant le nom de l'objet suivit d'un . et suivit du nom de la propriété.


var obj1 = {
    bool: false,
    nb: 20,
};
console.log(obj1.bool); // false
console.log(obj1.nb); // 20

Il est possible d'enchainer les propriétés de plusieurs objets.


var obj2 = {
    name: 'objet parent',
    obj: {
        name: 'objet enfant',
    },
};
console.log(obj2.obj.name); // 'objet enfant'

Méthode

On appelle méthode, une fonction qui est attachée à un objet.

Les méthodes ont un contexte qui correspond à l'environnement dans lequel elles sont appelées.

On accède à ce contexte avec la variable this.


function test() {
    return this.name;
}
var o1 = {
    name: 'objet 1',
    getName: test,
};
var o2 = {
    name: 'objet 2',
    getName: test,
};
console.log(o1.getName()); // 'objet 1'
console.log(o2.getName()); // 'objet 2'

Lors de la création de l'objet, nous n'avons passé que la variable test et nous n'avons pas exécuté la fonction à ce moment. C'est pourquoi il n'y a pas de () ici.

Bien qu'il s'agit de la même fonction, le résultat n'est pas le même.
Car dans le 1er appel, le context (this) vaut o1. Et dans le 2e il vaut o2.

Création d'une instance

On peut aussi créer des objets similaires qui suivent une même logique.

Il suffit d'utiliser le mot clef new devant l'appel d'une fonction.
Cette fonction fera office de constructeur.


function Example(value) {
    this.value = value;
    this.isPositive = value > 0;
}
var ex1 = new Example(7);
var ex2 = new Example(-5);
console.log(ex1.isPositive); // true
console.log(ex2.isPositive); // false

prototype

Lorsqu'un objet est crée à partir d'un constructeur, il bénéficie de toutes les valeurs du prototype de ce constructeur.


function Example(value) {
    this.value = value;
}
Example.prototype.limit = 10;
Example.prototype.isOverLimit = function() {
    return this.value > this.limit;

ex3.value = 12;
console.log(ex3.isOverLimit()); // true

map permet de recréer une autre liste à partir de chaque valeur de cette liste.

La fonction callback propose 3 arguments: la valeur, l'index et la liste sur laquelle on est en train d'itérer. Le résultat final dépend de ce que retourne cette fonction.

Base du HTML

Vu que le JavaScript s'exécute souvent dans les pages HTML. Une présentation rapide de ce langage s'avère nécessaire.


Le principe du HTML (Hyper Text Markup Language) est un langage descriptif, qui sert à mettre en page en décrivant les éléments de cette page.

Chaque élément est décrit par des tags <>. Et on peut les imbriquer pour décrire chaque sous-partie.

Les tags doivent être fermés en répétant le tag, précédé de/. Le contenu est situé entre ces 2 tags. Chaque tag peut avoir des attributs pour modifier son comportement.


<div>contenu</div>

<html>

<html> est l'élément de base. Tous les autres éléments seront placé à l'intérieur de lui et de ses enfants.

Il possède 2 enfants: <head> et <body>


Un seul élément est placé avant lui, un tag indiquant le type du document: <!doctype html> (pour indiquer que le document est décrit en HTML5).

<head>

<head> décrit les données générales du document.

Les 2 enfants principaux de <head> sont <meta> et <title>

<meta> décrit les meta-données du document. Principalement <meta charset="utf-8"> qui permet de dire que le document est encodé en UTF8. (ce tag ne se ferme pas il n'y a donc pas de </meta>)

<title> décrit le titre du document. Celui-ci est visible en haut de la fenêtre du navigateur ou dans l'onglet.


<head>
  <meta charset="utf-8">
  <title>Titre du document</title>
</head>

<body>

<body> va contenir tout ce qui s'affiche à l'écran.

Ses enfants seront affichés les uns après les autres selon leur ordre d'écriture.


<body>
  <div>un peu de texte.</div>
  <div>un autre contenu.</div>
</body>

À noter que dans le texte en HTML, tous les caractères blancs (espaces, tabulation, saut de lignes), sont remplacés par un seul espace.

<div> et <span>

<div> et <span> sont des éléments génériques sans auncun effet particulier ni aucune sémantique.

Ils servent souvent pour structurer le document.

La différence entre ces deux éléments est que <div> est un block alors que <span> est inline.

Un élément block essaiera de s'afficher sur toute la ligne et pourra être redimensionné.

Un élément inline s'intègre de le texte.

Il est déconseillé de mettre un élément block dans un élément inline.


<body>
  <div>un peu de texte.</div>
  <div>un autre contenu qui contient un autre <span>contenu</span>.</div>
</body>

<br>

<br> sert à afficher un saut de ligne.

<br> est une balise qui ne se ferme pas.



<div>
    text1
    text2
    <br>
    text3
</div>

text1 text2
text3

<script>

<script> permet d'exécuter du code JavaScript.

Il est fortement conseillé de mettre cette balise à la fin de <body> pour d'ordre de chargement et la performance.

Il est possible d'utiliser l'attribut src pour exécuter le code contenu dans un fichier. Ce qui est très recommandé pour des raisons de lisibilité et de maintenance.


<script>
console.log('code JavaScript exécuté');
</script>
                        

<script src="monCode.js"></script>

Remarque: si le chemin commence par # alors le lien redirige à l'intérieur de cette page (vers la balise qui possède un id correspondant au nom suivent ce caractère). En anglais cette partie est appelée hash.


<a href="https://google.com">Lien externe</a>
<a href="../index.html">Lien relatif à la page</a>
<a href="#/1">Lien interne</a>

  • type: le nom de l'event qui a été déclenché
  • target: l'élément qui a déclenché l'event
  • currentTarget: l'élément sur lequel l'écouteur était placé (il peut être différent de target)
  • key: la valeur de la touche qui a été utilisée

element.onchange = function (evt) {
    console.log(evt.type);
    evt.target.textContent = 'cliqué';
};

document

L'élément principal de la page s'appelle document. Il contient les éléments body et head.

Il est possible de lui ajouter des événements. Cela est pratique pour ajouter des événements claviers quand on veut juste écouter sur toutes la page (et pas lorsque le focus est sur un élément précis).


document.onkeydown = function(evt) {
    if (evt.key === 'enter') {
        console.log('la touche entrée a été pressée');
    }
};

Base du CSS

Le CSS (Cascading Style Sheet) permet de mettre en forme les éléments HTML.

On peut modifier le style d'un élément avec l'attribut style ou alors modifier le style de l'ensemble du document avec la balise <style></style>


<style>
    div {
        color: red;
    }
</style>
<div style="color: green">toto</div>

Structure du CSS

Le CSS est composé d'un ensemble de règles. Chaque règle les modifications à apporter aux éléments.

Une règle est composé d'un Selecteur et de propriétés.

Quand on utilise l'attribut style d'un élément HTML, on ne précise que les propriétés CSS (il n'y a pas de sélecteur).

Chaque propriété est séparée par un ;

Exemple


.une-classe {
    color: green;
    background-color: blue;
}

Résultat

Sélecteur

Les sélecteurs (ou selector en anglais) peremttent de dire quels sont les éléments sur lesquels il faudra appliquer les changements de valeur des propriétés.

Tous les éléments qui correspondent au selecteur seront impactés. Un élément peut être affecté par plusieurs règles (et une règle peut affecter plusieurs éléments).


Tous les sélecteurs CSS, peuvent être utilisés dans les fonctions document.querySelector et document.querySelectorAll qui servent à trouver des éléments HTML en JavaScript.


Nous ne présenterons ici que les instructions les plus simples.

Sélecteur: tag

Pour sélectionner un élément à partir de son type de balise, il suffit de mettre le nom de la balise dans le selecteur.


/* Tous les <div> s'écriront en jaune */
div {
    color: yellow;
}

/* Tous les <button> s'écriront avec un fond violet */
button {
    background-color: purple;
}

Un bouton qui se trouve dans un <div> s'écrira donc en jaune sur fond violet.

Sélecteur: class

Pour sélectionner un élément à partir d'une de ses classes, il suffit de mettre le nom de la classe préfixé par .


/* Tous les éléments ayant la classe erreur s'écriront en rouge */
.erreur {
    color: red;
}

/* Tous les éléments ayant la classe titre s'écriront deux fois plus gros */
.titre {
    font-size: 2em;
}

Sélecteur: id

Pour sélectionner un élément à partir de son id, il suffit de mettre le nom de l'id préfixé par #

Rappel: un même id ne peut peut être donné qu'à un seul élément dans la page.


/* L'élément ayant l'id user-name s'écrira en gras */
#user-name {
    font-weight: bold;
}

/* L'élément ayant l'id save-form s'écrira avec un fond orange */
#save-form {
    background-color: orange;
}

Sélecteur: combinaison d'instruction

Il est possible d'utiliser plusieurs instructions au sein d'un même sélecteur afin de rendre la recherche plus précise.


Quand toutes les instructions sont accolées, elles font références au même élément.


/* met en rouge seulement les <div> qui ont une classe important */
div.important {
    color: red;
}
/* met un fond bleu pour tous les éléments qui ont une classe field
   et une classe user */
.field.user {
    background-color: blue;
}

Sélecteur: éléments contenus

Les instructions séparées par un espace signifie une appartenance à un élément.


/* met en rouge les éléments ayant une classe important et contenu dans
   un élément <div> */
div .important {
    color: red;
}
/* met un fond bleu sur les éléments ayant une classe user et contenu
   dans un élément ayant une classe field */
.field .user {
    background-color: blue;
}

Il peut y avoir un ou plusieurs éléments entre l'élement sélectionné et son parent.

Sélecteur: éléments strictement contenus

Les instructions séparées par un > signifie une appartenance directe à un élément.

Il ne peut pas y avoir d'éléments entre l'élement sélectionné et son parent.


/* met en rouge les éléments ayant une classe important et
   enfant direct d'un élément <div> */
div > .important {
    color: red;
}
/* met un fond bleu sur les éléments ayant une classe user et
   enfant direct d'un élément ayant une classe field */
.field > .user {
    background-color: blue;
}

Propriétés CSS

Les propriétés CSS s'écrivent en deux parties séparées par un :

La première partie est le nom de la propriété.

La deuxième partie est la valeur de la propriété.


div {
     /* color est le nom de la propriété,
        blue est la valeur à affecter */
    color: blue;
    /* font-size est le nom de la propriété,
        20px est la valeur à affecter */
    font-size: 20px;
}

Il existe beaucoup de propriétés CSS, nous n'en verrons ici que quelques unes.

Valeur: color

Les valeurs de type color peuvent s'écrire de différente manière:

Manière Explication Exemples
color name Écrire directement le nom de la couleur (en anglais) orange, black, white, ...
code hexadécimal Écrire les valeurs hexadécimal des couleurs Rouge, Vert, Bleu préfixée par #. Chacune des couleurs va de 00 à FF. #FFCC00, #000000, #FFFFFF, ...
code rgb décimal Écrire les valeurs des couleurs Rouge, Vert, Bleu dans la fonction rgb. Chacune des couleurs va de 0 à 255. rgb(255, 60, 0), rgb(100, 200, 155), ...
code rgba Écrire les valeurs des couleurs Rouge, Vert, Bleu et alpha dans la fonction rgba. Chacune des couleurs va de 0 à 255. Alpha va de 0 à 1 (0 étant complétement transparent) rgba(255, 90, 0, 0.5), rgba(0, 0, 0, 1), ...
code hsl / hsla Écrire les valeurs de hub (couleur), saturation et luminance (et alpha) dans la fonction hsl (ou hsla). h varie de 0 à 60, s et l sont des pourcentages. Alpha va de 0 à 1 hsl(55, 50%, 70%), hsla(320, 60%, 40%, 0.6), ...

Valeur: Unités

Certaines valeurs (comme width ou height) s'expriment avec des unités.

unité Explication Exemple
px la valeur en pixel 12px
em une valeur relatif à la taille de la police 2em
% une valeur en pourcentage du parent 100%

color sert à définir la couleur du texte.

background-color sert à définir la couleur de fond de l'élément.



.with-color {
    color: #FF66FF;

.with-bg-color {
    background: rgb(255, 102, 255);

Quelques fonctions utiles

setTimeout

Sert à appeler une fonction plus tard.

Syntaxe


setTimeout(callback, time);
  • callback: une fonction qui sera exécutée dans "time" millisecondes
  • time: un nombre en milliseconde, indiquant la durée avant d'appeler la callback.


Exemple:


// affichage dans 2s
setTimeout(function() {
    console.log('affichage différé');
}, 2000);

clearTimeout

Pour arrêter un setTimeout, il faut utiliser clearTimeout.

clearTimeout prend en argument l'id du timeout qu'on veut arrêter.

L'id du timeout peut être récupérer par le retour de l'appel de la fonction setTimeout.


// démarrage de la fonction différée
var timerId = setTimeout(function() {
    console.log('code appélé plus tard');
}, 500);

// arrêt de l'appel différé
clearTimeout(timerId);

setInterval

Sert à appeler une fonction régulièrement et de manière répétée.

Syntaxe


setInterval(callback, time);
  • callback: une fonction qui sera exécutée toutes les "time" millisecondes
  • time: un nombre en milliseconde, indiquant la durée avant de répétée l'action.

Exemple:


// affichage toutes les 1/2s
setInterval(function() {
    console.log('affichage répétée');
}, 500);

// incrémente le comptage toutes les 3 secondes
var comptage = 0;
setInterval(function() {
    comptage = comptage + 1;
    console.log('Affichage répétée: ' + comptage);
}, 3000);

clearInterval

Pour arrêter un setInterval, il faut utiliser clearInterval.

clearInterval prend en argument l'id de l'interval qu'on veut arrêter.

L'id de l'interval peut être récupérer par le retour de l'appel de la fonction setInterval.


// démarrage de la répétition
var timerId = setInterval(function() {
    console.log('code qui se répète');
}, 500);

// arrêt de la répétition
clearInterval(timerId);

Créer un compteur/chronomètre

Objectif

  • Afficher le temps passé/restant
  • Faire évoluer le compteur
  • Créer des boutons permettant de démarrer et arrêter le chronomètre
  • Pouvoir configurer le chronomètre (sens du comptage, valeur initiale)

Afficher le temps

Il faudra une variable pour garder en mémoire la valeur du comptage (par exemple le nombre de secondes écoulées)

Il faudra créer une fonction pour convertir ce nombre de seconde en format hh:mm:ss.

Il faudra un espace où afficher ce résulat (et une fonction pour mettre à jour cet affichage).

Faire évoluer le compteur

Il faudra une fonction pour incrémenter/décrémenter la valeur de comptage toutes les secondes.

Il faudra une fonction pour arrêter ce comptage.

Ne pas oublier d'appeler la fonction d'affichage

Créer l'interface

Il faudra ajouter les boutons pour déclencher toutes les fonctions:

  • démarrage
  • arrêt
  • remise à zéro

Écouter des événements claviers, pour déclencher le chronomètre depuis des touches du clavier (comme espace)

Créer l'interface de paramétrage

Il faudra permettre de choisir une heure de départ

Il faudra permettre de changer de sens de comptage
